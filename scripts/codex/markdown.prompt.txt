== ARCHITECTURE YOU MUST FOLLOW ==
ivue owns observable state; OpenTUI owns terminal drawing/input; data flows one way: input -> model method -> mutation -> reactive invalidation -> requestRender.

Three class kinds:
- Reactive domain model (observable, identity, disposable): the DEFAULT for stateful models.
- Plain class (algorithms/resources, non-reactive): parsers, transports, process handles, engines.
- Static capability class (stateless backend ops): file/process/git command wrappers.

ivue facts (from project.decisions.md — do not get these wrong):
- ivue's ONLY public export is `Reactive()`. `ref`/`shallowRef`/`computed`/`watch`/`watchEffect` come from `vue` (already installed). Import reactivity from 'vue', wrappers from 'ivue'.
- `Static()` and the kernel are NOT in the ivue package (vendored elsewhere in the app). If your module needs a static capability, write it as a normal class with static methods for now (a plain `namespace X { const $Class=$X; let Class=$X }`), do not import Static from ivue.

The namespace pattern (EXACT shape for every class file, PascalCase.ts):
```ts
import { Reactive } from 'ivue';
import { ref, shallowRef } from 'vue';
class $Thing {
  get items() { return shallowRef<Row[]>([]); }   // ref-returning getter = mutable reactive state (.value)
  get count() { return this.items.value.length; }  // PLAIN getter = cheap derived state (reactive, 0 bytes/instance) — DO NOT use computed() for this
  add(r: Row) { this.items.value = [...this.items.value, r]; }
  dispose() { /* close handles first */ (this as any).$stopEffects?.(); }
}
export namespace Thing {
  export const $Class = $Thing;             // raw, immutable, children `extends` this
  export let Class = Reactive($Class);       // mutable selection slot; consumers do `new Thing.Class()`
  export type Model = InstanceType<typeof Class>;
  export type Instance = typeof Class.Instance;
}
```
Rules: cross-module deps read LATE via getters (`get Dep(){ return Dep.Class }`) — never top-level `new ImportedClass()` or `const C = X.Class`. Long-lived effects use `this.$watch`/`this.$watchEffect` and an owner calls `this.$stopEffects()` on dispose. Plain classes use the same namespace but `let Class = $Class` (no Reactive wrapper). Role-collection files (not one class) are named <module>.<role>.ts. NO HTTP roles (no controllers/routes/services).

Performance invariants that bind your module:
- Cost tracks the OBSERVED set: high-cardinality data (git commit history, markdown tokens, diagnostics) uses compact records/typed arrays + virtualized visible rows + eviction — NEVER one reactive object per commit/token/line.
- Async results (git refresh, LSP responses, parses) can outlive the state they describe: stamp every async result with a revision/version and DISCARD stale ones; the immediate UI must NEVER block on a subprocess.
- LSP/git are separate failable processes: lazy start, dispose on close, degrade gracefully if absent/slow/crashed. Editing must never freeze or crash on their failure.
