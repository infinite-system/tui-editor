MODULE: lsp   (brief milestone M5; TypeScript LSP client stack, multi-provider)
Files (write under src/modules/lsp/):
- JsonRpc.ts — plain class: LSP JSON-RPC framing (Content-Length) encode/decode over a byte stream; correlate request/response by id; handle partial + multiplexed messages in one chunk.
- LspTransport.ts — plain class: stdio transport over a child process stdin/stdout using JsonRpc.
- LspProcess.ts — plain class: spawn/own an LSP subprocess (Bun.spawn), lifecycle, dispose (kill+close). Never throw into the editor on spawn failure — expose an error state.
- LanguageProvider.ts — interface/types: id, command+args resolution, capabilities.
- TypeScriptProvider.ts — implements LanguageProvider; order vtsls -> typescript-language-server (detect availability; no hard-fail if absent).
- LanguageClient.ts — Reactive model. LAZY start (only when a semantic feature is requested / TS-JS file opens). initialize handshake; publishDiagnostics REVISION-STAMPED (tie to document version, discard stale); definition; hover + references if straightforward. LSP positions are UTF-16 — convert at the boundary. Disposable: shut server down on dispose. Survives server absence/crash without crashing the editor.
Load-bearing: lazy + disposable + revision-stamped + survives-failure. Do NOT wire into the editor; expose a clean client API. Tests: JSON-RPC framing (encode/decode, partial + multiplexed chunks); lifecycle (lazy start, dispose kills process) using a FAKE in-process scripted stdin/stdout server (do not require a real LSP binary); stale-diagnostic discard by version.
